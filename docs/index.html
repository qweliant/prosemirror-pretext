why this one's different .

            
not "another rich-text editor" — a different rendering substrate

            

                

                    
🚫 No contenteditable

                    
Zero DOM text nodes. Every glyph is one ctx.fillText(). None of the
                        cross-browser contenteditable selection/IME/paste chaos that every other editor fights.

                    
vs. Slate / TipTap / Lexical — all contenteditable under the hood.

                

                

                    
🌊 Float-wrap, both sides

                    
Drag a photo anywhere and text reflows around it instantly — wrapping on both sides of an
                        obstacle, recomputed every drag frame.

                    
CSS Shapes can't: float-only, one side, no live line geometry.

                

                

                    
🦻 Canvas you can still read

                    
Canvas is normally invisible to assistive tech. We keep a hidden, screen-reader-visible DOM mirror
                        in sync — open “screen-reader view” above to see exactly what it reads.

                    
Most canvas editors are an accessibility dead end.

                

                

                    
⚡ Flat input latency

                    
Pretext line-breaks with pure math, so reading the caret position each keystroke is a
                        cache lookup, not a reflow — and stays flat as the document grows. The
                        “0 reflows” under the editor isn't marketing; it's the architecture.
                    

                    
DOM editors pay a read-after-write reflow that scales with the page — ~11× slower
                        at 8k blocks. see the benchmark →

Marks & nodes

                    
Bold, italic, code, links, underline, strike, super/subscript, text color, highlight; headings,
                        blockquote, code blocks, rules.

                

                
📋
                    
Lists & alignment

                    
Nested bullet/ordered lists with markers + indent, Tab/Shift-Tab nesting, and left/center/right
                        block alignment.

                

                
🖼️
                    
Images

                    
Block node views; drag the corner to resize, drag the body to float so text wraps around — the
                        same obstacle model Pretext's demos use.

                

                
⌨️
                    
Real editing

                    
Grapheme-aware caret, gap cursor between blocks, hard breaks, rich HTML paste, undo/redo, and a
                        familiar prosemirror-view-shaped API.

                

                
🔒
                    
Read-only mode

                    
Toggle editable — navigation, selection, and copy keep working while
                        edits are dropped. (Try the button above.)

                

                
🧩
                    
Bring your schema

                    
Peer-deps the ProseMirror stack; you define the schema, marks, and node views. Float-wrap and
                        styling are opt-in per node.

                

            

        

    


    
    

        

            
quick start 🍵

            
a peer of the ProseMirror stack — bring your own schema

            
bun add prosemirror-pretext

// canvas, all the way down 🐸
import { CanvasEditor, markSpecs, buildMarkKeymap } from 'prosemirror-pretext'
import { EditorState } from 'prosemirror-state'

await document.fonts.ready // fonts must load before layout

const editor = new CanvasEditor({
  state: EditorState.create({ doc, schema }),
  container: document.querySelector('#editor'),
  font: '16px Georgia, serif',
  lineHeight: 26, // set it ~1.4–1.6× your font — it's not derived from `font`
  ariaLabel: 'My notes',
})
